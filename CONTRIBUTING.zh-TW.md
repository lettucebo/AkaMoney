# 為 AkaMoney 做出貢獻

感謝您有興趣為 AkaMoney 做出貢獻！本文件提供貢獻的指南和說明。

[English](CONTRIBUTING.md) | 繁體中文

## 開始使用

1. Fork 儲存庫
2. 複製您的 fork：`git clone https://github.com/YOUR_USERNAME/AkaMoney.git`
3. 建立新分支：`git checkout -b feature/your-feature-name`
4. 進行更改
5. 徹底測試您的更改
6. 提交您的更改：`git commit -am '新增某些功能'`
7. 推送到分支：`git push origin feature/your-feature-name`
8. 建立 Pull Request

## 開發設定

按照 [SETUP.zh-TW.md](SETUP.zh-TW.md) 指南設定您的開發環境。

## 程式碼風格

### TypeScript/JavaScript
- 使用 TypeScript 以確保型別安全
- 遵循 ESLint 規則（如已配置）
- 使用有意義的變數和函數名稱
- 為複雜邏輯新增註解
- 保持函數小而專注

### Vue 元件
- 使用 `<script setup>` 語法
- 使用 Composition API
- 保持元件專注和可重用
- 使用 props 和 emits 進行元件通訊

### CSS
- 儘可能使用 Bootstrap 類別
- 保持自訂 CSS 最少
- 在 Vue 元件中使用作用域樣式
- 遵循行動優先方法

## 提交訊息

遵循約定式提交格式：
- `feat:`：新功能
- `fix:`：錯誤修復
- `docs:`：文件更改
- `style:`：程式碼風格更改
- `refactor:`：程式碼重構
- `test:`：測試新增/更改
- `chore:`：維護任務

範例：`feat: 為短網址新增 QR 碼生成`

## 測試

在提交 PR 之前：
1. 在本地測試您的更改
2. 測試前端和後端
3. 測試邊緣情況和錯誤情境
4. 確保沒有控制台錯誤
5. 在不同瀏覽器上測試（Chrome、Firefox、Safari）
6. 測試行動裝置上的響應式設計

## Pull Request 指南

1. **清楚的描述**：解釋您做了什麼更改以及為什麼
2. **引用問題**：連結到相關問題（例如：「修復 #123」）
3. **截圖**：包含 UI 更改的截圖
4. **測試**：如適用，新增或更新測試
5. **文件**：如需要，更新 README.md 或其他文件
6. **清理提交**：如有必要，壓縮提交
7. **無合併衝突**：在主分支上重新定基

## 功能請求

要請求新功能：
1. 檢查 Issues 中是否已請求
2. 建立新 issue 並使用「enhancement」標籤
3. 描述功能及其使用案例
4. 解釋為什麼它有價值

## 錯誤報告

要報告錯誤：
1. 檢查 Issues 中是否已報告
2. 建立新 issue 並使用「bug」標籤
3. 包含：
   - 錯誤描述
   - 重現步驟
   - 預期行為
   - 實際行為
   - 截圖（如適用）
   - 環境（瀏覽器、作業系統、Node 版本）

## 程式碼審查流程

1. 維護者將審查您的 PR
2. 處理任何回饋或請求的更改
3. 批准後，維護者將合併您的 PR
4. 您的貢獻將包含在下一個版本中
